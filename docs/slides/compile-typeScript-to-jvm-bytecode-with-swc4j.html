Motivation

The Problem

  • Running TypeScript on JVM requires a JavaScript runtime
  • V8 (via Javet), GraalJS, Node.js — all bring native dependencies
  • Runtime startup overhead, memory cost, version management
  • Gap between script world and Java world — no shared type system
Motivation

The Idea

TypeScript's syntax is close to Java —
classes, interfaces, type annotations, inheritance.

What if we compile it directly to JVM bytecode?

Demo

Hello Bytecode

Java
ByteCodeCompiler compiler = ByteCodeCompiler.of(
    ByteCodeCompilerOptions.builder()
        .jdkVersion(JdkVersion.JDK_17)
        .build());

ByteCodeRunner runner = compiler.compile("""
    export function add(a: int, b: int): int {
      return a + b
    }""");

int result = runner
    .createStaticRunner("$")
    .invoke("add", 1, 2);
Output
1 + 2 = 3
  • No V8 involved
  • Pure JVM execution
  • Top-level function → static method

Classes and Namespaces

TypeScript
namespace com {
  export class Calculator {
    add(a: int, b: int): int {
      return a + b
    }
    multiply(a: double, b: double): double {
      return a * b
    }
  }
}
Java invocation
runner.createInstanceRunner("com.Calculator");
classRunner.invoke("add", 3, 4);
classRunner.invoke("multiply", 2.5, 4.0);
Output
add(3, 4) = 7
multiply(2.5, 4.0) = 10.0

namespace → Java package
export class → public JVM class

Type System & Inference

TypeScript
export function demo(): String {
  const x = 5          // inferred: int
  const y = 10         // inferred: int
  const label = "sum"  // inferred: String
  return label + ": " + (x + y)
}
Output
sum: 15

Type Mapping

  • int, long, float, double → JVM primitives
  • String → java.lang.String
  • Integer, Long → auto boxing/unboxing
  • Literals inferred: 5 → int, 3.14 → double
  • String + → StringBuilder chain

Control Flow

TypeScript
export function classify(n: int): String {
  if (n > 0) return "positive"
  else if (n < 0) return "negative"
  else return "zero"
}

export function factorial(n: int): int {
  let result: int = 1
  for (let i: int = 2; i <= n; i = i + 1)
    result = result * i
  return result
}
Output
classify(42) = positive
classify(-7) = negative
classify(0) = zero
factorial(5) = 120
  • if / else if / else
  • while, for, do-while
  • switch, try/catch/finally
  • Recursion
Killer Feature

Java Interop

TypeScript
namespace com {
  export class MyList
      extends java.util.ArrayList<Object> {
    label: String = "my-list"
    getLabel(): String {
      return this.label
    }
  }
}
Java
ArrayList<Object> list =
    (ArrayList<Object>) classRunner.getInstance();
list.add("Hello");
list.add("World");
Output
label = my-list
size = 2
items = [Hello, World]
  • Real JVM subclass of ArrayList
  • Cast, instanceof, reflection all work
  • Inherited methods: add(), size()
  • Custom fields and methods

TypeScript Interfaces → Java Interfaces

TypeScript
export interface Greeter {
  name: String
  greet(): String
}
export class SimpleGreeter
    implements Greeter {
  name: String = ""
  constructor(name: String) {
    this.name = name
  }
  getName(): String { return this.name }
  setName(name: String): void { this.name = name }
  greet(): String { return "Hello, " + this.name }
}
Output
Greeter is interface: true
SimpleGreeter implements Greeter: true
greeting = Hello, Alice
  • Properties → abstract getter/setter
  • Interface extends (single + multiple)
  • Standard Java reflection confirms it
Summary

What's Supported

  • Classes, fields, constructors
  • Inheritance (extends, super)
  • Interfaces (implements)
  • Abstract classes
  • Method overloading
  • Constructor overloading + chaining
  • Default parameters
  • Static fields and methods
  • Control flow (if/while/for/switch/try)
  • Type inference
  • Java interop (extend ArrayList, etc.)
  • TypeScript interfaces → Java interfaces
  • TypeScript enums → Java enums
  • Nested namespaces
  • Arrow functions
  • Template literals
